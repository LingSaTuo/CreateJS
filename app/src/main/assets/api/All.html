CJS库的机制

>>总章

不要使用 const 关键词定义过于普遍的全局变量或常量 ，当冲突时会抛出异常 TypeError:redeclaration of const xxx

>>库的结构

文件夹结构类似下方

/库名字
/库名字/libs.propres(务必包含)
/库名字/文件1.js
/库名字/文件2.js
/库名字/文件夹/文件.js
/库名字/API/api.html（该库的API文档）

如 ：

Common
Common/Debug.js
Common/A/文件.js
Common/libs.propres

libs.propres文件

该文件在库的根目录下，如果不在根目录下CJS将不识别该库，libs.propres 遵循json格式 ，用法样例如下

{
     //必选 （库内文件列表，排序决定加载顺序）
    "include":[{"name":"Debug.js"},
                    {"name":"A/文件.js"}],
//可选 （依赖库）当该库正常工作需要依赖其他库时，就需要添加此键值对
    "libs":[{"name":"A","uri":"当你知道A库的下载直链，需要填入直链"},
                {"name":"B","uri":"true 库在本地时，填入true即可，无论是在js_libs下还是user_libs下"}]，
//可选（作者信息）
    "info":[{"The_creator":"作者名字",
                "Correction":"修改者（优化者）",
                "for":"服务平台"}],
//可选（检查更新 code为int 类型，updateuri为校验文件直链）
    "versioncode":[{"code":1,"updateuri":"校验文件直链"}]
}
校验文件

校验文件遵循json格式，当versioncode大于本地code时，会提示更新

{
"versioncode":2,
"message":"更新内容",
"downloaduri":"下载直链"
}

>>库机制

CJS会优先读取libs.propres文件，从中获取目标库所要加载的文件，以及目标库的信息，依赖，版本等属性

函数调用（可选但推荐）

该函数，推荐但不强制，CJS调用函数和调试功能是基于Common库（DeBUG库），该库在CJS库工坊下载，你也可以内置函数调用函数，但函数名必须为_function 

function _function(str){
//TODO
}

其中参数为 函数名!"参数1","参数2","参数3"，例如： 调用了 function a(str){} _function参数会是"a"!""

载入完成回掉

该函数为可选，不强制 函数名必须为loadingfinish

function loadingfinish(str){

}

传入的参数为加载的库名称数组 如 :["Common"]
